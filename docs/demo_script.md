# 6—8 分钟演示与答辩脚本

## 角色

- 主讲：控制浏览器，讲用户问题、主流程和最终价值。
- 后端答辩：解释 MySQL、任务幂等、不可变版本、LangGraph 检查点和恢复。
- AI 答辩：解释 RAG、内部自动生成能力、结构化输出、确定性审校和最多两轮返修。
- 质量保障：计时，准备备用项目和 PPT，演示故障恢复与自动检查。

## 时间线

**0:00—0:40，问题与边界。** 主讲说明系统面向小学教师单课时备课，不替代教师决策；最终只交付教师确认后的 PPT。

**0:40—1:30，模板、项目与资料。** 输入《水的三态变化》课程信息，展示系统推荐和五个已验证模板；预览后选择“活力探索”，确认创建时才下载主题。随后上传公开样例，展示解析/索引状态和检索结果。

**1:30—2:15，生成设置。** 进入具体项目的工作台，选择参考资料并补充课堂要求；说明系统会在后台自动调用所需能力。

**2:15—3:20，完整生成。** 使用同一资料生成完整 PPT。展示简洁的生成进度，检查 40 分钟流程、逐页讲解提示和三级练习已经自动进入课件。

**3:20—4:10，主题与图片。** 展示生成结果如何使用项目模板支持的封面、章节、重点和活动版式；上传一张本地图片，选择当前页和右侧位置，预览立即显示。

**4:10—5:05，局部修改与版本。** 选择一页要求“精简页面文字”，保存新版本；展示 `changed_ids` 仅含目标页、其他块哈希不变。切换历史版本，再回滚并说明回滚也创建新版本。

**5:05—5:55，自动质量工作流。** 说明 LangGraph 七节点在后台自动执行，审校与最多两轮返修不占用教师操作界面。

**5:55—6:40，PPT 导出。** 在项目工作台点击“导出 PPT”，说明渲染器复用创建项目时已经确认并缓存的固定版本主题。使用 PowerPoint 或 WPS 打开 `.pptx`，展示主题样式、课件图片和讲解备注。

**6:40—7:20，故障恢复。** 展示无密钥时明确的降级标记；说明网络错误可重试、版本冲突返回 409 和当前版本、服务重启会把悬挂任务标为可重试失败，GraphRun 可从检查点恢复。

**7:20—8:00，证据与收尾。** 展示测试结果、Docker 健康状态、两种分辨率截图和三组课例记录。总结：可追溯、可修改、可恢复、可安全交付。

## 高频答辩

- 为什么 MySQL 和 Chroma 都需要？MySQL 保证业务事务和版本一致性；Chroma 只保存可重建的检索索引。
- 如何防止 AI 编造引用？检索结果生成固定 Citation；无证据时只允许通用建议，结构校验不通过则修复或降级。
- 为什么局部修改安全？前端提交基础版本和稳定块 ID；后端乐观锁、只合并目标块，并比较未变块哈希。
- LangGraph 是否真的运行？完整任务和手动启动接口都会调用编译后的条件图；节点事件、尝试次数、问题和状态快照落 MySQL。
- 如何避免无限返修？设计、课件、讲解提示和练习各节点最多自动尝试两次，再由后台结束本次检查。
- 为什么只有 PPT？教师的实际课堂载体就是课件，因此教学设计、活动、讲解备注、练习与解析统一装入一个文件，减少重复操作。
- 为什么直接生成 PPTX？教师可以在常用的 PowerPoint 或 WPS 中继续修改，也可以直接用于课堂展示；内部结构化数据仅用于生成、审校和版本控制。
