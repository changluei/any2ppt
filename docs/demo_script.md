# 6—8 分钟演示与答辩脚本

## 角色

- 主讲：控制浏览器，讲用户问题、主流程和最终价值。
- 后端答辩：解释 MySQL、任务幂等、不可变版本、LangGraph 检查点和恢复。
- AI 答辩：解释 RAG、五个 Skill、结构化输出、确定性审校和最多两轮返修。
- 质量保障：计时，准备备用项目和 ZIP，演示故障恢复与学生包隐私。

## 时间线

**0:00—0:40，问题与边界。** 主讲说明系统面向小学教师单课时备课，不替代教师决策；四类产物和所有 AI 草案均需教师确认。

**0:40—1:30，项目与资料。** 新建《水的三态变化》项目，上传公开样例，展示解析/索引状态和检索结果；点击引用说明文件、位置、分块和摘录可追溯。

**1:30—2:15，独立 Skill。** 运行“学情与教学目标设计”，在任务卡展开独立结构化结果；强调这不是完整备课包，也不会覆盖其他产物。

**2:15—3:20，完整生成。** 使用同一资料生成完整备课包。展示任务阶段、进度、取消/重试、trace_id；切换教学设计、课件、讲稿、练习，检查 40 分钟流程、16 页课件、逐页讲稿和三级练习。

**3:20—4:10，课件与引用。** 用缩略页切换，展示安全 iframe 渲染和 Markdown 源码；指出讲稿随页同步、来源点击可回到原文。

**4:10—5:05，局部修改与版本。** 选择一页要求“精简页面文字”，保存新版本；展示 `changed_ids` 仅含目标页、其他块哈希不变。切换历史版本，再回滚并说明回滚也创建新版本。

**5:05—5:55，质量工作流。** 展示 LangGraph 七节点、节点尝试次数和审校问题。点击问题定位内容；选择定向返修或人工确认。说明自动返修最多两轮，仍失败就停在人审。

**5:55—6:40，双包导出。** 为每类产物选择版本，导出教师包和学生包。打开文件清单：教师包有讲稿/答案/引用，学生包没有讲稿、答案、解析；`slides.md` 可交给 Slidev，`slides.html` 可离线预览和打印 PDF。

**6:40—7:20，故障恢复。** 展示无密钥时明确的降级标记；说明网络错误可重试、版本冲突返回 409 和当前版本、服务重启会把悬挂任务标为可重试失败，GraphRun 可从检查点恢复。

**7:20—8:00，证据与收尾。** 展示测试结果、Docker 健康状态、两种分辨率截图和三组课例记录。总结：可追溯、可修改、可恢复、可安全交付。

## 高频答辩

- 为什么 MySQL 和 Chroma 都需要？MySQL 保证业务事务和版本一致性；Chroma 只保存可重建的检索索引。
- 如何防止 AI 编造引用？检索结果生成固定 Citation；无证据时只允许通用建议，结构校验不通过则修复或降级。
- 为什么局部修改安全？前端提交基础版本和稳定块 ID；后端乐观锁、只合并目标块，并比较未变块哈希。
- LangGraph 是否真的运行？完整任务和手动启动接口都会调用编译后的条件图；节点事件、尝试次数、问题和状态快照落 MySQL。
- 如何避免无限返修？设计、课件、讲稿/练习各节点最多自动尝试两次，然后进入人工确认。
- 学生包怎样避免泄露？导出服务按白名单重建学生练习，移除答案和解析，且从不选择讲稿产物。
- 为什么不直接生成 PPTX？首版选择 Slidev 兼容 Markdown，便于审阅、版本比较和安全渲染；离线 HTML 可打印 PDF，PPTX 可作为后续独立渲染能力。
